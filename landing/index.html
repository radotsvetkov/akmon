AKMON

Prove what any AI agent did. Offline, with no lock-in.

Akmon is a tamper-evident evidence and verification layer for AI agents. It sits on top of whatever agent you already run, through OpenTelemetry or with Akmon's own reference agent, and turns every session into a portable, signed record. A third party can verify a signature offline with nothing but openssl. No Akmon install, no cloud.

// why akmon

The evidence is
the product

Akmon does not replace your agent. It sits on top of it and turns each session into a sealed record that someone else can verify independently, with standard tools, even on a machine that never had Akmon installed.

🔌

Sits on top of any agent

Import any OpenTelemetry GenAI trace with akmon otel import. It reads the v1.37 structured form and the older v1.36 message-event form that most agents still emit.

🔏

Tamper-evident and signed

Each session is a content-addressed, hash-linked record. akmon bundle sign adds an offline Ed25519 signature over the head, so a record is attributable to a key.

🔍

Verify offline with openssl

akmon bundle prove-openssl writes a statement, signature, and public key. A stranger checks the signature with plain openssl, no Akmon and no cloud.

🧰

Standalone verifier for auditors

agef-verify is a small, separate binary. An auditor can confirm integrity and authorship without installing the full Akmon toolchain.

🪪

Operator identity

akmon bundle attest records the accountable person behind a session, signed separately from the head, so a record carries who, not just what.

📐

Honest capture levels

Own-agent runs are full and replay deterministically. Imported traces are structural and never dressed up as full recordings. The level is signed into the record.

📦

Portable AGEF bundles

Evidence exports as a single content-addressed AGEF bundle (v0.1.3) you can hand off, archive, and verify anywhere. Signatures and attestations are optional add-ons.

⚙️

Single Rust binary

Runs in local shells, SSH, and CI with no plugin runtime stack. Deterministic akmon replay is available for sessions Akmon produced itself.

// the producer-agnostic pipeline

From any agent to an offline proof

Make a key, import any agent's OpenTelemetry trace, export a portable bundle, sign it, verify it, and emit a proof a stranger can check with plain openssl. The verification chain is the same whether the session came from Akmon's own agent or another tool. Akmon v2.2.0 implements AGEF v0.1.3.

akmon bundle keygen --out signer.pk8 --public-out signer.pub
akmon otel import trace.json --journal ./journal
akmon bundle export <session-id> --journal ./journal --output audit.akmon
akmon bundle sign audit.akmon --key signer.pk8
akmon bundle verify audit.akmon --verify-key signer.pub --require-signature
akmon bundle prove-openssl audit.akmon --verify-key signer.pub --out-dir proof
openssl pkeyutl -verify -pubin -inkey proof/pubkey.pem -rawin -in proof/statement.bin -sigfile proof/signature.bin

// get started

Install in seconds

GitHub Releases attach real akmon and agef-verify binaries per platform, plus a SHA256SUMS file. Writing to /usr/local/bin usually requires sudo on macOS, so installing to ~/bin avoids that. If a download is HTML instead of a binary, the matching release asset is missing (cut a v* tag after the release workflow runs).

# macOS or Linux with Homebrew. The tap is live.
brew tap radotsvetkov/akmon
brew install akmon # the CLI
brew install agef-verify # the standalone verifier for auditors

akmon --version # akmon 2.2.0
# Verifying with openssl needs OpenSSL 3.x: brew install openssl@3
# Apple Silicon: M1, M2, M3, M4 (no sudo)
mkdir -p ~/bin
curl -L https://github.com/radotsvetkov/akmon/releases/latest/download/akmon-darwin-arm64 \
  -o ~/bin/akmon && chmod +x ~/bin/akmon
# The standalone verifier ships on every release too:
curl -L https://github.com/radotsvetkov/akmon/releases/latest/download/agef-verify-darwin-arm64 \
  -o ~/bin/agef-verify && chmod +x ~/bin/agef-verify
# If `akmon` is not found, add ~/bin to PATH (zsh):
# echo 'export PATH="$HOME/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc

akmon --version # akmon 2.2.0
akmon config # run setup wizard
# Import any agent's OTEL trace, then sign and verify it
akmon otel import trace.json --journal ./journal
mkdir -p ~/bin
curl -L https://github.com/radotsvetkov/akmon/releases/latest/download/akmon-darwin-x86_64 \
  -o ~/bin/akmon && chmod +x ~/bin/akmon
# The standalone verifier for auditors:
curl -L https://github.com/radotsvetkov/akmon/releases/latest/download/agef-verify-darwin-x86_64 \
  -o ~/bin/agef-verify && chmod +x ~/bin/agef-verify

akmon --version # akmon 2.2.0
mkdir -p ~/bin
curl -L https://github.com/radotsvetkov/akmon/releases/latest/download/akmon-linux-x86_64 \
  -o ~/bin/akmon && chmod +x ~/bin/akmon
# The standalone verifier for auditors:
curl -L https://github.com/radotsvetkov/akmon/releases/latest/download/agef-verify-linux-x86_64 \
  -o ~/bin/agef-verify && chmod +x ~/bin/agef-verify

akmon --version # akmon 2.2.0
# Install both binaries straight from the repo:
cargo install --git https://github.com/radotsvetkov/akmon akmon
cargo install --git https://github.com/radotsvetkov/akmon agef-verify

# Or build from a clone:
git clone https://github.com/radotsvetkov/akmon
cd akmon
# Slim build: 3.4MB, no semantic indexing
cargo build --release --no-default-features
# Full build: 20MB, with semantic indexing
cargo build --release

mkdir -p ~/bin
cp target/release/akmon ~/bin/
# 1. Install Ollama from https://ollama.com
ollama pull qwen2.5-coder:7b

# 2. Install Akmon (example: Apple Silicon to ~/bin)
mkdir -p ~/bin
curl -L https://github.com/radotsvetkov/akmon/releases/latest/download/akmon-darwin-arm64 \
  -o ~/bin/akmon && chmod +x ~/bin/akmon

# 3. Open any project
cd my-project
akmon chat # free, offline, private
