AKMON TOOL REFERENCE (for the model)
=====================================

This section documents every built-in Akmon tool: JSON arguments, successful outputs, and errors.
Paths are always relative to the project sandbox root unless noted. Absolute paths and ".." segments are rejected.

GENERAL OUTPUT SHAPE
------------------
Success: {"status":"success","content":"<string>"}  (content may embed JSON as text)
Error: {"status":"error","code":"<snake_case>","message":"<string>"}

---
TOOL: list_directory
--------------------
Purpose: List files and subdirectories at a path (use before read_file on unknown trees).

Arguments (JSON object):
  path (string, required): Relative directory path; use "." for project root.

Example invocation:
  {"path":"."}

Typical success: content lists entry names (files and dirs) with types.
Typical errors: not_found, path_escape if outside sandbox.

---
TOOL: read_file
---------------
Purpose: Read full UTF-8 text from one file (capped by max bytes, default 1 MiB).

Arguments:
  path (string, required): Relative path to file.

Example:
  {"path":"crates/akmon-core/src/lib.rs"}

Success: file body as string in content (may be truncated with notice if huge).
Errors: not_found, not_a_file (if path is directory), binary_content, path_escape, too_large.

---
TOOL: write_file
----------------
Purpose: Atomically write UTF-8 text to a file (new file or replace). Usually requires policy confirmation.

Arguments:
  path (string, required)
  content (string, required)

Example:
  {"path":"notes.txt","content":"hello"}

Errors: invalid_args, path_escape, permission_denied (policy), io errors as permission_denied or message.

---
TOOL: search
------------
Purpose: Regex search across text files under a path (exact/pattern matching). Prefer this for symbols/strings.

Arguments:
  pattern (string, required): Rust regex pattern.
  path (string, optional): Directory or file root; default ".".
  file_pattern (string, optional): Glob filter e.g. "*.rs".
  case_sensitive (boolean, optional): default false.

Example:
  {"pattern":"fn main","path":"crates","file_pattern":"*.rs"}

Success JSON (inside content string) includes matches with paths and lines; may set truncated:true when capped.
Errors: invalid_args for bad regex, path_escape.

---
TOOL: edit
----------
Purpose: Replace exactly one occurrence of old_str with new_str in a file (unique match required).

Arguments:
  path (string, required)
  old_str (string, required)
  new_str (string, required; may be empty to delete old_str)

Example:
  {"path":"src/lib.rs","old_str":"pub const OLD: i32 = 1;","new_str":"pub const OLD: i32 = 2;"}

Errors: not_found, ambiguous_match (if old_str appears 0 or >1 times), path_escape.

---
TOOL: patch
-----------
Purpose: Apply a unified diff to one or more files.

Arguments:
  patch (string, required): Full unified diff body (---/+++ / @@ lines).

Errors: patch_failed, not_found, path_escape.

---
TOOL: web_fetch (only if registered)
------------------------------------
Purpose: HTTPS GET public URL; SSRF-blocked for private IPs and metadata hosts.

Arguments:
  url (string, required): https URL only.

Errors: permission_denied, invalid_args for disallowed URL.

---
TOOL: shell (only if --shell-allow patterns registered)
-------------------------------------------------------
Purpose: Run allowlisted argv-only commands (no shell metacharacters).

Arguments:
  command (string, required): Must match a configured allowlist pattern.

Errors: subprocess_failed, permission_denied.

---
TOOL: semantic_search (only with CLI --index)
---------------------------------------------
Purpose: Natural-language codebase search using embeddings. Use for concepts ("error handling policy").
For exact strings or regex, use search.

Arguments:
  query (string, required)
  top_k (integer, optional): default 5, max 20.

Returns JSON with query, results: [{file, start_line, end_line, score, preview}, ...].
If index is still building: error message "indexing in progress". If embedding model failed to load: unavailable message.

---
WORKFLOW HINTS
--------------
1) list_directory(".") then search or read_file.
2) Prefer edit/patch over write_file for existing files.
3) Never read_file on a directory.
